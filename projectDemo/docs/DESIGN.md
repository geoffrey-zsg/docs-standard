# DESIGN.md

## 1. 文档目的

统一设计系统与前端实现之间的约定，让人和 AI 都能按同一套视觉和交互语言工作。

## 2. 设计原则

- 一致性优先于局部创意
- 信息层级清晰
- 交互反馈明确
- 异常状态可感知
- 响应式行为可预测

## 3. 设计令牌

填写项目实际值（CSS 变量，详细定义见 `docs/frontend/styling.md`）：

- 主色：`var(--color-primary)` → 待填写（示例：`#3b82f6`）
- 危险色：`var(--color-danger)` → 待填写（示例：`#ef4444`）
- 正文色：`var(--color-text)` → 待填写（示例：`#1f2937`）
- 字号基准：正文 `1rem`，辅助 `0.875rem`，标题 `1.25rem`
- 间距基准：`--space-4` → `1rem`（8px 栅格）
- 圆角：`--radius-md` → 待填写（示例：`0.5rem`）
- 动效：过渡 `150ms ease`，不超过 `300ms`

> Shadcn/UI 项目：实际 token 由 `globals.css` 中的 CSS 变量定义，初始化后补充实际值。

要求：
- 所有视觉值优先引用 token
- 不在业务组件里硬编码品牌色和间距值

## 4. 组件规范

使用 Shadcn/UI 组件库，基础组件通过 CLI 安装后放入 `src/shared/ui`。

每个业务组件说明：
- 适用场景
- 变体（来自 Shadcn/UI 或自定义扩展）
- 禁止用法
- 交互反馈

关键组件约定：
- **按钮**：变体主要/次要/幽灵/危险；加载中显示 spinner 并禁用重复点击；一个弹窗主操作区禁止出现两个主要按钮
- **表格**：筛选条件固定置顶；空状态提供下一步动作；行操作超过 3 个收进更多菜单
- **表单**：提交态、禁用态、错误提示必须完整；不允许无反馈的异步提交

## 5. 页面布局规范

- 页面头部：标题 + 主要操作按钮
- 筛选区：条件吸顶，支持重置
- 主内容区：表格/卡片/详情
- 操作区：批量操作、导出等
- 详情侧栏 / 弹层：不影响背景层的操作

## 6. 状态设计

每种状态都有统一视觉与文案策略，不允许只做成功态而不做异常态：

- **加载中**：Skeleton 或 Spinner，禁用可能触发重复请求的操作
- **空状态**：说明为何为空 + 下一步引导操作
- **错误状态**：给出可执行的错误信息，不只说"失败"
- **禁用状态**：灰化 + Tooltip 说明原因
- **成功状态**：Toast 反馈，不替代表单内错误提示

## 7. 响应式与适配

- 断点：`sm: 640px / md: 768px / lg: 1024px / xl: 1280px`
- 表格窄屏：横向滚动或卡片化展示
- 弹窗移动端：全屏或底部 Sheet
- 文本溢出：单行截断 + Tooltip 展示完整内容

## 8. 文案与反馈

- 操作文案明确动词和对象（"删除订单"而非"删除"）
- 错误提示给出可执行信息（"手机号格式不正确"而非"输入错误"）
- 危险操作二次确认
- Toast 不替代表单内错误提示
