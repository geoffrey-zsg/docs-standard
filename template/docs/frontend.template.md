## 核心约束

### 组件开发
- 优先复用现有组件，不重复造基础组件
- 基础组件放共享层，业务组件放业务模块内
- 一个组件一个职责，复杂逻辑抽到 hooks

### 样式规则
- 统一使用设计令牌（Design Tokens）
- 禁止硬编码颜色、间距、字号
- 不混用多套样式方案

### 状态管理
- UI 局部状态放组件内
- 跨页面共享状态放全局状态层
- 筛选/分页/排序等可分享状态优先放 URL

### 数据请求
- 请求统一通过共享 API 客户端（如 `src/shared/api`）
- 接口类型优先从契约生成（OpenAPI/GraphQL）
- 页面不直接拼接请求路径
- 所有请求处理 loading、error、empty 三态

### Mock 边界定义
**允许 Mock 的部分**：
- 后端 API 响应数据
- 第三方服务调用
- 复杂计算结果

**即便 Mock 也必须完整的交互**：
- 上传状态反馈（进度、成功、失败）
- 错误态展示和处理
- 流式输出效果（如打字机效果）
- 引用跳转和联动
- 表单校验和提交流程
- 加载态和禁用态切换

**Mock 实现要求**：
- Mock 数据结构必须与真实 API 一致
- 异步行为必须模拟（延迟、错误概率）
- 边界情况必须可测试（空数据、超长文本、特殊字符）

### 表单规则
- 使用统一表单库和 schema 校验
- 表单提交态、禁用态、错误提示必须完整
- 不允许无反馈的异步提交

## 安全规范

### 输入验证与防护
- **XSS 防护**：
  - 所有用户输入必须转义后再渲染
  - 使用框架内置的安全机制（React 的 JSX 自动转义）
  - 禁止使用 `dangerouslySetInnerHTML`（除非有充分理由并经过审查）
  - 富文本内容必须使用 DOMPurify 等库清洗

- **SQL 注入防护**：
  - 前端表单输入必须校验格式
  - 后端必须使用参数化查询（前端无法完全防护，但要配合）

- **CSRF 防护**：
  - 敏感操作必须验证 CSRF Token
  - 使用 SameSite Cookie 属性

### 敏感信息处理
- **禁止硬编码**：
  - API Key、密钥、Token 不得写入代码
  - 使用环境变量管理敏感配置
  - 生产环境配置不得提交到代码仓库

- **数据脱敏**：
  - 手机号显示：`138****5678`
  - 身份证显示：`110***********1234`
  - 银行卡显示：`6222 **** **** 1234`

- **敏感操作确认**：
  - 删除、转账、权限变更等操作必须二次确认
  - 关键操作需要输入密码或验证码

### 权限控制
- **前端权限校验**：
  - 根据用户角色显示/隐藏功能
  - 无权限时禁用按钮并提示
  - 注意：前端校验仅为用户体验，安全依赖后端

- **路由守卫**：
  - 未登录用户重定向到登录页
  - 无权限用户重定向到 403 页面

### 依赖安全
- 定期检查依赖漏洞：`pnpm audit`
- 及时更新有安全漏洞的依赖
- 避免使用不维护的过时依赖

### 日志安全
- 前端日志不得包含敏感信息（密码、Token、身份证号）
- 生产环境关闭详细错误堆栈输出
- 错误上报前脱敏处理

## 禁止事项
- 禁止组件内直接请求后端
- 禁止复制粘贴已有组件后另起一套
- 禁止跳过类型错误直接提交
- 禁止硬编码敏感信息（API Key、密钥、Token）
- 禁止使用 `dangerouslySetInnerHTML` 而不做内容清洗
- 禁止在前端日志中输出敏感信息
