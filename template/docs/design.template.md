# DESIGN.md

## 1. 文档目的

本文档用于统一设计系统与前端实现之间的约定，让人和 AI 都能按同一套视觉和交互语言工作。

## 2. 设计原则

- 一致性优先于局部创意
- 信息层级清晰
- 交互反馈明确
- 异常状态可感知
- 响应式行为可预测

## 3. 设计令牌

填写项目实际值（CSS 变量，详细定义见 `docs/frontend/styling.md`）：

- 主色：`var(--color-primary)` → 示例 `#3b82f6`
- 危险色：`var(--color-danger)` → 示例 `#ef4444`
- 正文色：`var(--color-text)` → 示例 `#1f2937`
- 字号基准：正文 `1rem`，辅助 `0.875rem`，标题 `1.25rem`
- 间距基准：`--space-4` → `1rem`（8px 栅格）
- 圆角：`--radius-md` → 示例 `0.5rem`
- 动效：过渡 `150ms ease`，不超过 `300ms`

要求：

- 所有视觉值优先引用 token
- 不在业务组件里硬编码品牌色和间距值

## 4. 组件规范

- 按钮
- 输入框
- 选择器
- 弹窗 / 抽屉
- 表格
- 空状态
- 反馈提示 / Toast

每个组件建议说明：

- 适用场景
- 变体
- 禁止用法
- 交互反馈

## 5. 页面布局规范

- 页面头部
- 筛选区
- 主内容区
- 操作区
- 详情侧栏 / 弹层

## 6. 状态设计

- 加载中
- 空状态
- 错误状态
- 禁用状态
- 成功状态

要求：

- 每种状态都有统一视觉与文案策略
- 不允许只做成功态，不做异常态

## 8. 文案与反馈

- 操作文案明确动词和对象
- 错误提示给出可执行信息
- 危险操作二次确认
- Toast 不替代表单内错误提示

## 9. 样例

```md
## 设计令牌
- 主色：`var(--color-brand-600)`
- 危险色：`var(--color-danger-600)`
- 圆角：`var(--radius-md)`
- 间距：`var(--space-4)`

## 按钮
- 变体：主要 / 次要 / 幽灵 / 危险
- 加载中状态下显示加载指示器并禁用重复点击
- 禁止在一个弹窗主操作区同时出现两个主要按钮

## 表格
- 筛选条件固定置顶
- 空状态提供下一步动作
- 行操作超过 3 个时收进更多菜单
```
